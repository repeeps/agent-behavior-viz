PART 1 · FINITE STATE MACHINE

상태 스택 — 되돌아갈 곳을 기억하다

일반 FSM은 "지금 어떤 상태인지"만 알 뿐, 어디서 왔는지는 모르죠. 일시정지 후 정확히 직전 화면으로 돌아오려면 상태 스택이 필요해요. 상태를 push로 쌓고 pop으로 꺼내는 "스택을 가진 상태 기계"를 직접 눌러 봅시다. (전문 용어로는 푸시다운 오토마타라 부르는데, 이름은 몰라도 돼요 — 핵심은 "상태를 차곡차곡 쌓아 기억한다".)

Step 00 — Remember Where to Return

되돌아갈 곳을 기억해야 할 때

일반 FSM의 전이는 "다음 상태가 어디인지"가 미리 고정돼 있어요. 그런데 일시정지처럼 "하던 걸 멈췄다가 정확히 그 자리로" 돌아가야 할 땐 문제가 생기죠 — 멈추기 직전이 어떤 상태였는지 FSM은 기억하지 못하거든요.

FSM의 한계

"게임플레이 중 일시정지했다"고 해봐요. 일시정지에서 재개하면 게임플레이로 돌아가야 해요. 그런데 만약 메뉴 화면에서도 일시정지할 수 있다면? 일반 FSM은 재개 시 어디로 갈지 화살표를 하나로 못 정해요.

스택이 주는 능력

멈추기 직전 상태를 스택에 쌓아두면(push), 재개할 때 맨 위를 꺼내(pop) 정확히 그리로 복귀해요. 스택은 곧 "상태의 기억"이에요 — 어디서 왔는지를 LIFO(후입선출) 순서로 보관하죠.

→ 다음 스텝에서 실제 배열 스택으로 push / pop 을 눌러 봅니다.

Step 01 — push / pop

일시정지는 push, 재개는 pop

게임플레이 중 일시정지하면 pause 상태를 스택에 push해요. 재개하면 pause를 pop해서 아래에 깔려 있던 게임플레이로 정확히 복귀하죠. 스택 맨 위가 곧 현재 상태예요.

상태 스택 (맨 위 = 현재)

→ push는 위에 쌓고, pop은 맨 위만 꺼내요(LIFO). 바닥의 게임플레이는 절대 사라지지 않아서, 몇 번을 일시정지·재개해도 항상 같은 자리로 돌아와요.

Step 02 — Nested Menus

중첩 메뉴: 깊이 들어갔다 정확히 되돌아오기

게임 → 설정 → 키설정 으로 들어갈 때마다 화면을 push해 스택이 깊어져요. 뒤로를 누르면 pop하며 직전 화면으로 한 단계씩 정확히 복귀하죠. 깊이가 어떻게 변하는지 보세요.

화면 스택 · 깊이 1

→ 들어간 순서의 정확히 역순으로 빠져나와요. 키설정에서 뒤로 가면 설정, 또 뒤로 가면 게임. 어느 깊이에서든 "직전이 무엇이든 그리로" 돌아가는 게 스택의 힘이에요.

Step 03 — Stack vs. Plain FSM

FSM과의 차이: 고정 전이 vs. 기억된 복귀

일반 FSM

전이 대상이 고정

"이 상태에서 나가면 항상 저 상태로." 화살표가 미리 박혀 있어요.

상태 스택

직전으로 복귀

"이전이 무엇이든 그리로." 어디서 왔는지를 스택이 기억해요.

정체

상태의 기억

스택 = 거쳐온 상태들의 역순 기록. push/pop으로만 접근(LIFO).

한 문장 요약

일반 FSM은 "일시정지 → 게임플레이"처럼 복귀 대상을 하나로 못 박지만, 스택은 push해둔 그 무엇이든 pop으로 정확히 되살려요. 그래서 스택은 상태의 기억이에요.

⚠ 한계 — 스택만으로는 "걸으면서 동시에 총 쏘기" 같은 여러 행동의 동시 진행을 다루기 어려워요(한 번에 맨 위 하나만 활성).
