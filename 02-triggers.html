PART 1 · FINITE STATE MACHINE

트리거 — 전이를 일으키는 두 종류

상태와 상태를 잇는 화살표(전이)는 저절로 당겨지지 않아요. 무언가 방아쇠(트리거)를 당겨야 하죠. 트리거는 크게 둘이에요 — 밖에서 들어오는 이벤트(키 입력·클릭)와, 상태 변수가 임계를 넘는 조건(체력<20). 직접 눌러보고 슬라이더를 움직이며 차이를 느껴봅시다.

Step 00 — What Pulls the Trigger

전이는 무엇이 일으키나

전이를 당기는 방아쇠는 두 종류예요. 이벤트는 밖에서 들어오는 순간적 사건, 조건은 상태 변수를 지속적으로 살피다 임계를 넘으면 발동해요.

이벤트 Event

밖에서 들어오는 사건

키 누름, 마우스 클릭, 메시지 도착처럼 그 순간에만 존재하는 외부 입력. 들어오면 한 번 전이를 시도하고 사라져요.

예: 점프 키, 버튼 클릭, "문 열어"

조건 Condition

상태 변수가 임계를 넘음

체력·거리·시간 같은 변수를 계속 평가하다, 임계를 넘는 순간 자동으로 전이해요. 아무 입력 없이도 값만 변하면 발동하죠.

예: 체력≤20, 적과의 거리<5

한 문장 요약

이벤트는 "~가 일어났다"(사건), 조건은 "~인 상태다"(평가). 이벤트는 눌러야 오고, 조건은 값만 바뀌어도 와요.

Step 01 — Event Trigger

이벤트 트리거: 버튼을 눌러 전이

아래는 자동문 FSM이에요. 버튼(이벤트)을 누를 때만 전이가 일어나요. 현재 상태에서 정의된 이벤트만 작동하고, 정의 없는 이벤트는 무시돼요.

→ 핵심은 능동성이에요. 이벤트는 누군가(사용자·다른 시스템)가 보내야 도착해요. 보내지 않으면 상태는 영원히 그대로죠 — 자동문은 버튼을 누르기 전까지 잠긴 채 기다려요.

Step 02 — Condition & Guard

조건 트리거 + 가드

슬라이더만 움직여 보세요. 버튼을 누르지 않아도 체력이 임계를 넘으면 상태가 자동으로 바뀌어요. 가드(guard)는 "조건이 참일 때만" 전이를 허용하는 잠금장치예요.

80

→ 전이 규칙: 체력 ≤30이면 위급으로, ≥31이면 정상으로. 가드(조건)가 참이 되는 순간 입력 없이 전이해요. 조건 트리거는 "끊임없이 지켜보는 감시자"예요 — 슬라이더를 내려 임계를 통과시켜 보세요.

Step 03 — Event vs Condition

둘의 차이: 한 FSM에 섞어보기

실전 FSM은 둘을 함께 써요. 아래 경비병은 이벤트(소리 들림·놓침)와 조건(체력≤20 자동 도망)을 모두 가져요. 버튼도 눌러보고, 슬라이더도 내려보세요.

주체외부가 보냄 (push)내부가 매 틱 평가 (polling)

→ 이벤트는 "사건이 났다", 조건은 "그런 상태다". 위 표의 경비병에서 "소리 들림"은 이벤트(눌러야 옴), "체력≤20 도망"은 조건(값만 내려도 옴)이에요. 좋은 설계는 둘을 적재적소에 섞어요.
