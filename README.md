# 행동 설계, 눈으로 보기 (FSM & Behavior Tree, Visualized)

게임 NPC·로봇·UI는 "지금 무엇을 할지"를 어떻게 정할까요? 두 대표 방식 **유한 상태 기계(FSM)** 와 **행동 트리(Behavior Tree)** 를, 상태 그래프와 tick 애니메이션으로 직접 돌려보며 익히는 인터랙티브 학습 자료입니다. 모든 상태 전이·tick 평가가 브라우저에서 실제 로직으로 동작합니다.

> 직접 만든 **비영리 학습용 자료**입니다. 특정 교재·강의와 무관하며, 설명·예시·코드는 모두 독자적으로 제작했습니다.

빌드·의존성 없음. `index.html` 을 브라우저로 열면 끝입니다.

```bash

python3 -m http.server 8000     # → http://localhost:8000
```

## 챕터 (13개)

**PART 1 · 유한 상태 기계 (FSM)**
| # | 주제 | 핵심 인터랙션 |
|---|------|--------------|
| 01 | 상태와 전이 | 신호등·캐릭터 — 이벤트로 상태 점등·전이 |
| 02 | 트리거: 이벤트 vs 조건 | 입력 이벤트(push)와 조건 폴링·가드 |
| 03 | FSM으로 NPC 만들기 | 순찰→추격→공격→도망 (거리·체력 슬라이더) |
| 04 | 계층적 FSM (HFSM) | 상태 그룹화·중첩·공통 전이 |
| 05 | 상태 스택 (Pushdown) | 일시정지·중첩 메뉴 push/pop 복귀 |
| 06 | FSM의 한계 | 전이 N×(N−1) 폭발 → BT로 가는 다리 |

**PART 2 · 행동 트리 (Behavior Tree)**
| # | 주제 | 핵심 인터랙션 |
|---|------|--------------|
| 07 | Tick과 3가지 상태 | 루트부터 순회 · Success/Failure/Running 점등 |
| 08 | Sequence (AND) | 순서 실행·하나 실패 시 중단·Running 전파 |
| 09 | Selector / Fallback (OR) | 하나 성공 시 중단 · 우선순위 |
| 10 | Decorator | Inverter·Repeater·Cooldown |
| 11 | Blackboard | 트리가 공유하는 데이터, 한 값이 여러 노드에 |
| 12 | BT로 같은 NPC | 03의 NPC를 BT로 — 매 tick 재평가하는 반응성 |

**PART 3 · 비교**
| # | 주제 | 핵심 인터랙션 |
|---|------|--------------|
| 13 | FSM vs Behavior Tree | 같은 NPC 나란히 · 확장성 비교 · 언제 무엇을 |

## 설계 원칙

- **실제 로직**: 상태 전이·tick 단축평가가 모두 실제 코드로 동작 (하드코딩 결과 없음)
- **같은 NPC, 두 방식**: 03(FSM)·12(BT)·13(비교)에서 동일한 NPC를 두 패러다임으로 만들어 대조
- **일관된 시각 언어**: 상태 점등(현재=앰버), tick 색(성공=초록·실패=빨강·진행중=노랑)
